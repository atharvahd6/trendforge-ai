# Agent Guide — TrendForge Venture Lab

Read this first if you're an AI coding assistant (Claude Code, Cursor, Copilot, etc.)
picking up this repo. Then read `PROJECT_CONTEXT.md` for the design-decision history.

## What this repo does

One Python script (`agent.py`), no framework, run once per day (cron / GitHub Actions):
research an idea → write it as HTML → audit it → deploy it to `products/<date>-<slug>/` →
log it in `MASTER_TREND_JOURNAL.md` → regenerate `index.html` from that journal.

## Rules before you touch anything

1. **Don't hardcode a single provider into a pipeline stage.** That's exactly the bug this
   repo just had (audit hardcoded to Mistral — see README "Known Issues"). Every stage should
   resolve its provider from `strategy` + which `*_API_KEY` env vars actually exist, the way
   `resolve_audit_provider()` does.
2. **Slugs must stay date-prefixed.** `products/<slug>/` alone caused silent overwrites of a
   previous day's live tool when two titles collapsed to the same 6-word slug. Never remove
   the `today_str` prefix or the collision-suffix while-loop in `run_agent_pipeline()`.
3. **The research prompt must keep excluding recent titles.** `get_recent_titles(limit=15)` +
   the "do not repeat" clause is what stops the scout converging on one comfortable idea for
   weeks at a time. If you rewrite the research prompt, keep that constraint.
4. **`final_html` must always be raw HTML, no code fences.** Every stage prompt says
   "output ONLY the raw HTML" for a reason — `strip_code_fences()` is a safety net, not the
   primary defense. Keep the explicit instruction in any prompt you edit.
5. **`manual_ideas.txt` must be cleared after a manual run**, or every subsequent run will
   keep re-consuming the same manual idea instead of resuming autonomous scouting.
6. **`MASTER_TREND_JOURNAL.md` is the single source of truth for `index.html`.** Never hand-edit
   `index.html` directly — it's fully regenerated by `rebuild_index_html()` from the journal
   every run. Hand edits will be silently overwritten on the next run.

## Where things live

| Concern | File |
|---|---|
| Provider wrappers | `agent.py` → `call_gemini` / `call_groq` / `call_mistral` |
| Fallback ordering | `agent.py` → `FALLBACK_STRATEGIES` |
| Audit provider resolution | `agent.py` → `resolve_audit_provider()` |
| Slug / collision handling | `agent.py` → `run_agent_pipeline()`, the `while os.path.exists(...)` loop |
| Anti-repetition | `agent.py` → `get_recent_titles()` |
| Ledger schema | `MASTER_TREND_JOURNAL.md` (markdown table: Date Run \| Concept \| Link) |
| Dashboard template | `agent.py` → `render_index_html()` |

## Before you ship a change

- Run `python -m py_compile agent.py` at minimum.
- If you touch `run_agent_pipeline()`, do a dry run with a throwaway `manual_ideas.txt` entry
  and confirm: (1) a new dated folder appears under `products/`, (2) the journal gets exactly
  one new row, (3) `index.html` regenerates with that row included, (4) `manual_ideas.txt` is
  blank afterward.
- Don't add a new required environment variable without updating the startup credential check
  at the top of `agent.py` — right now it deliberately requires only *one* of the three keys.
